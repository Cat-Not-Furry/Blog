Contribuir

Todo el código público del sitio y de los proyectos enlazados sigue una filosofía open-source. Aquí tienes formas concretas de ayudar.

Empezar

Los repositorios viven bajo GitHub · Cat-Not-Furry. Puedes:

  • Proponer mejoras o correcciones con pull requests.
  • Abrir issues para bugs, ideas o dudas de uso.
  • Mejorar README, comentarios o guías sin tocar lógica crítica.

Ver organización en GitHub

Código y revisiones

Antes de enviar un PR, conviene abrir un issue breve o comentar en uno existente para alinear el cambio. Los parches pequeños y bien explicados son más fáciles de integrar.

  • Un cambio = una idea (cuando sea posible): más simple de revisar y revertir.
  • Incluye pasos para probar o capturas si afecta a UI o flujos visibles.
  • Respeta el estilo del archivo que toques (indentación, naming, sin reformateos masivos no relacionados).
git checkout -b fix/descripcion-corta

Comunidad

Si usas herramientas para fighting games, Linux o scripts de automatización, compartir experiencia (qué distro, qué stick, qué falló) en un issue ayuda mucho a quien llegue después.

Las contribuciones no tienen por qué ser solo código: traducciones, ejemplos de configuración o avisos de enlaces rotos en la web también cuentan.

Licencia

Los proyectos mostrados en este sitio están bajo licencias libres; en la página de cada repo encontrarás el archivo LICENSE concreto. El pie del sitio indica GPL v3 como referencia general para el material publicado aquí.

Al contribuir, aceptas que tu aporte se distribuya bajo la misma licencia que aplique al repositorio correspondiente.
